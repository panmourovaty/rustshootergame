# Map Format - RustShooterGame

This document describes how to create a map package for RustShooterGame and how to serve it to players via the dedicated server.

---

## Overview

Maps are distributed as a **`.tar.zst`** file (a `tar` archive compressed with Zstandard).  
The server operator passes the HTTPS URL of this file to the server binary via `--map-url`.  
When a client connects, the server immediately sends the URL via the network protocol.  
The client downloads the archive, extracts it into memory, and loads the 3D assets through Bevy's asset pipeline.

---

## Quick Start

```bash
# 1. Build your map files (see below).

# 2. Create the archive.
tar -cf mymap.tar scene.glb textures/ spawn_points.txt
zstd -19 mymap.tar -o mymap.tar.zst

# 3. Host the file on any HTTPS server (e.g. S3, Cloudflare R2, nginx, etc.)
#    and note the public URL.

# 4. Run the dedicated server with the map URL.
./server --map-url https://example.com/maps/mymap.tar.zst
```

---

## Archive Layout

```
mymap.tar.zst
├── scene.glb           ← REQUIRED - visual geometry, materials, lights + collision
├── textures/           ← OPTIONAL - external textures (if not embedded in GLB)
│   ├── floor_albedo.png
│   ├── wall_normal.png
│   └── …
└── spawn_points.txt    ← OPTIONAL - player spawn locations
```

All paths inside the archive are resolved relative to the archive root.  
Forward slashes are used regardless of the platform that created the archive.

---

## `scene.glb` - Visual Scene + Collision

This file is loaded as both the **visual representation** and the **physics collision** of the map.  
Every mesh in the scene automatically receives a trimesh collider - no separate collision file is needed.

### What to include

| Element | Notes |
|---|---|
| Meshes & materials | Standard PBR metallic-roughness (Base Color, Metallic, Roughness, Normal map) |
| Embedded textures | Preferred - embed textures in the GLB to avoid separate files |
| Lights | Optional - point, spot, and directional lights are supported |
| Sky / environment | Optional - can be a background mesh or sky dome |

> **Collision is automatic.** Every mesh in `scene.glb` gets a trimesh collider generated by Avian3D.
> You do not need a separate `collision.glb`.  If you want a mesh to be visible but non-solid,
> remove it from the GLB or place it in a separate object with no collision (not currently configurable - all meshes collide).

See the [Blender Authoring Guide](#blender-authoring-guide) below for step-by-step export instructions.

### Coordinate system

Bevy uses a **right-handed Y-up** coordinate system, the same as glTF 2.0.  
Blender is Z-up; the glTF exporter converts automatically when **Y Up** is checked (it is on by default).

---

## Physics collision

Collision is generated automatically from `scene.glb` - every mesh receives a **trimesh collider** after the GLTF scene is instantiated.  No extra file is needed.

### Performance tips

- **Keep meshes reasonably low-poly.** Trimesh colliders use the actual triangle data, so high-poly decorative geometry (railings, foliage, fine detail) adds physics cost without gameplay benefit.  Consider using simplified box/capsule proxy meshes for complex shapes.
- **Close all meshes.** Gaps or open edges can cause players to fall through at high speeds.  In Blender use **Select → All by Trait → Non-Manifold** to find and fix open edges.
- **Separate large surfaces.** A single enormous mesh for the entire floor is fine; many tiny disconnected fragments can be slower to query.

## Blender Authoring Guide

This section covers everything needed to produce GLB files that load correctly in Bevy 0.19.

### Prerequisites

- **Blender 3.3 LTS or newer** (4.x recommended).  The built-in glTF 2.0 exporter is used; no add-ons are required.
- Keep **Unit Scale at 1.0** (the default) and model in **metres**.  One Blender unit = one metre in-game.  If you modelled at a different scale, apply it before export: select all, **Ctrl+A → All Transforms**.

---

### Understanding the coordinate system

| Axis | Blender | Bevy / glTF |
|---|---|---|
| Right | +X | +X |
| Forward | +Y | −Z |
| Up | +Z | +Y |

The glTF exporter re-maps axes automatically when **Y Up** is enabled.  **Do not rotate the scene manually** to compensate; let the exporter do it.

---

### Setting up materials (PBR metallic-roughness)

Bevy's GLTF loader reads the **glTF metallic-roughness PBR** material model.  Blender's **Principled BSDF** shader exports to this model automatically.

**Required node setup:**

```
Image Texture (Base Color) ──► Principled BSDF ──► Material Output
Image Texture (ORM)        ──►   (Metallic, Roughness inputs)
Image Texture (Normal Map) ──► Normal Map node ──► Normal input
```

**Principled BSDF inputs that export to glTF:**

| Principled BSDF input | glTF property | Notes |
|---|---|---|
| Base Color | `baseColorFactor` / `baseColorTexture` | sRGB color or texture |
| Metallic | `metallicFactor` | 0 = dielectric, 1 = metal |
| Roughness | `roughnessFactor` | 0 = mirror, 1 = fully rough |
| Normal | `normalTexture` | Must go through a **Normal Map** node set to **Tangent Space** |
| Emission Color | `emissiveFactor` / `emissiveTexture` | For glowing surfaces |
| Alpha | `baseColorTexture` alpha channel | Set blend mode on material (see below) |

> **Nodes that do NOT export:** Subsurface Scattering, Specular, Sheen, Coat, IOR.  They are silently ignored.  Keep materials simple - Principled BSDF with the inputs above.

**Texture color spaces - set these correctly or colors will be wrong:**

| Texture type | Blender color space |
|---|---|
| Base Color (albedo) | **sRGB** |
| Normal map | **Non-Color** |
| Roughness / Metallic / ORM | **Non-Color** |
| Emissive | **sRGB** |

Set color space in the Image Texture node's **Color Space** dropdown.

**ORM texture (packed channel map):**  
Pack Occlusion (R), Roughness (G), Metallic (B) into a single PNG.  Connect the individual channels using a **Separate Color** node:

```
Image Texture (ORM, Non-Color)
    ├── R → Ambient Occlusion (not connected to Principled BSDF - AO is a separate pass)
    ├── G → Roughness
    └── B → Metallic
```

**Alpha / transparency:**

Set the material's **Blend Mode** (Properties → Material → Settings → Blend Mode):
- `Opaque` - fully solid (default, best performance)
- `Alpha Clip` - binary cutout transparency (fences, foliage)
- `Alpha Blend` - smooth transparency (glass, water)

---

### Texture recommendations

| Property | Recommended format | Notes |
|---|---|---|
| Base Color | PNG (with alpha) or JPEG (no alpha) | JPEG is smaller; PNG needed for alpha cutout |
| Normal map | PNG | Never JPEG - compression artefacts corrupt normals |
| ORM | PNG | Lossless keeps channel data accurate |
| Emissive | JPEG or PNG | JPEG fine if no alpha needed |

- Keep textures at **powers of two** (512, 1024, 2048, 4096).  Non-power-of-two sizes work but may not generate mipmaps correctly on all drivers.
- **2048×2048** is a good default for large surfaces.  Use **1024×1024** for small props.
- Embed textures in the GLB for simplicity (set **Images → Export** to **Automatic** in the export dialog).

---

### Exporting `scene.glb`

**File → Export → glTF 2.0 (.glb/.gltf)**

Go through each tab in the export dialog and set the following:

#### Format tab

| Setting | Value |
|---|---|
| Format | **GLB** (Binary - single file, no separate .bin or texture files) |

#### Include tab

| Setting | Value |
|---|---|
| Selected Objects | Off (export everything) or On (selected only) |
| Visible Objects | **On** - hides objects in hidden collections are excluded |
| Active Collection | Off |
| Custom Properties | Off |
| Cameras | Off (not used by the game) |
| Punctual Lights | **On** if your scene has Blender lights (Point, Spot, Sun) |

#### Transform tab

| Setting | Value |
|---|---|
| Y Up | **On** ✓ (enabled by default - converts Blender Z-up to glTF Y-up) |

> **Do not change this.** Turning it off will rotate the entire scene 90° in-game.

#### Data → Mesh tab

| Setting | Value |
|---|---|
| Apply Modifiers | **On** - bakes all modifiers into the exported mesh |
| UVs | **On** |
| Normals | **On** |
| Tangents | **On** - required for normal maps to display correctly |
| Vertex Colors | On if you use vertex colors; Off otherwise |
| Loose Edges / Points | Off |
| **Compression** | **OFF - never enable this.** Draco (`KHR_draco_mesh_compression`) is not supported by Bevy's GLTF loader and will cause a load error. |

#### Data → Material tab

| Setting | Value |
|---|---|
| Materials | **Export** |
| Images → Format | **Automatic** (PNG for textures with alpha, JPEG for opaque) |
| Images → Quality | 85–90 for JPEG |
| Images → Export | **Automatic** (embeds textures in the GLB) |

> If you prefer external texture files, set **Images → Export** to **None** and place the textures in the `textures/` directory of your archive.

#### Animation tab

Static maps do not need animations.  Leave the defaults or uncheck **Animation** entirely.

---

### Before you export - checklist

Run through this in Blender before hitting **Export**:

1. **Apply all transforms** - Select all, **Ctrl+A → All Transforms**.  Unapplied scale can cause normals to flip and physics colliders to be the wrong size.
2. **Apply all modifiers** - Or leave **Apply Modifiers** on in the export dialog.
3. **Check UV maps** - Every mesh that has a texture must have a UV map.  Bevy uses UV channel 0 (the first UV map in Blender).
4. **Check color spaces** - Normal maps and ORM textures must be **Non-Color**; Base Color must be **sRGB**.
5. **No negative scale** - Negative scale inverts normals and breaks physics.  Fix with **Ctrl+A → Scale**.
6. **No N-gons on collision meshes** - Blender triangulates on export, but complex N-gons can triangulate unpredictably.  Use Ctrl+T (Triangulate Faces) on collision meshes before export to control the result.
7. **Lights** - If the scene has no lights, it will appear black in-game.  Add at least one directional (Sun) light.

---

### Collision geometry tips

Because every mesh in `scene.glb` gets a trimesh collider, keep these in mind:

- **High-poly decorative objects** (fences, railings, foliage) will generate expensive trimesh colliders.  For such objects, consider replacing their geometry with a simple box or cylinder before export, or accept the performance cost if they are small.
- **Open meshes** (planes, flat quads used as decals) generate one-sided colliders.  Players can pass through them from the back.  Use closed volumes (even thin boxes) for floors and walls.
- **Use Select → All by Trait → Non-Manifold** in Blender to identify meshes with boundary edges before export.

### What Bevy's GLTF loader supports

Use this table to know what you can and cannot do in your material setup:

| Feature | Supported | Notes |
|---|---|---|
| PBR metallic-roughness | ✅ | Standard glTF material model |
| Base Color texture | ✅ | PNG or JPEG |
| Normal map (tangent space) | ✅ | Must use Normal Map node in Blender |
| Roughness / Metallic texture | ✅ | Packed in G / B channels of a single texture |
| Occlusion texture | ✅ | R channel |
| Emissive texture | ✅ | |
| Alpha Opaque / Mask / Blend | ✅ | Set Blend Mode on the Blender material |
| Double-sided materials | ✅ | Enable **Backface Culling** off in Blender material settings |
| Vertex colors | ✅ | |
| Multiple UV channels | ⚠️ | Only UV0 is used for textures |
| Punctual lights (Point, Spot, Sun) | ✅ | Requires `KHR_lights_punctual` - enabled by Blender when **Punctual Lights** is checked |
| KTX2 / Basis Universal textures | ✅ | Smaller GPU memory; requires KTX2 tools to prepare |
| Draco mesh compression | ❌ | **Not supported.** Causes a load error. Never enable. |
| Specular / Sheen / Coat materials | ❌ | glTF extensions not implemented in Bevy 0.19 |
| Transmission / Volume | ❌ | Not supported |
| Morph targets / blend shapes | ❌ | Loaded but not animated |
| Skeletal animation | ⚠️ | Loaded but the game does not play animations on map geometry |

---

### Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Map fails to load, error mentions `KHR_draco_mesh_compression` | Draco compression was enabled on export | Re-export with **Compression off** |
| Map loads but is completely black | No lights in the scene | Add a Sun light in Blender and re-export with **Punctual Lights on** |
| Textures appear pink / missing | Texture path not found or texture was not embedded | Re-export with **Images → Export → Automatic** to embed, or add files to `textures/` |
| Normal map looks wrong / inverted | Wrong color space or missing Normal Map node | Set image color space to **Non-Color**; connect via a **Normal Map** node set to **Tangent Space** |
| Geometry is inside-out / normals flipped | Negative scale or flipped normals | Select all in Blender, apply scale (**Ctrl+A → Scale**), then **Mesh → Normals → Recalculate Outside** |
| Physics collider is wrong size | Unapplied scale on collision mesh | Apply scale (**Ctrl+A → Scale**) before export |
| Players fall through the floor | Gap in collision mesh | Use **Select → All by Trait → Non-Manifold** to find and fix open edges |
| Scene rotated 90° in-game | **Y Up** was turned off in export | Re-export with **Transform → Y Up: On** |
| Transparent surfaces opaque | Blend Mode not set | Set material **Blend Mode** to **Alpha Blend** or **Alpha Clip** in Blender |

## `textures/` - External Textures (optional)

If you chose **not** to embed textures inside the GLB, place them here and reference them with relative paths in your GLTF JSON.

```
textures/
├── floor_albedo.png        ← Base color / albedo
├── floor_normal.png        ← Normal map (OpenGL convention, Y-up)
├── floor_orm.png           ← Occlusion (R) / Roughness (G) / Metallic (B)
└── wall_albedo.png
```

The GLTF file should reference them as `../textures/floor_albedo.png` (relative to `scene.gltf`, if you use the non-binary `.gltf` format) or simply `textures/floor_albedo.png` relative to the archive root.

> **Tip:** Embedding textures in the `.glb` avoids any path resolution issues and is recommended for most maps.

## `spawn_points.txt` - Player Spawn Locations (optional)

A plain-text file with one spawn point per line.  
Each line must contain three space-separated floating-point numbers: **X Y Z**.

```
-15.0 2.0 -15.0
 15.0 2.0  15.0
-15.0 2.0  15.0
 15.0 2.0 -15.0
```

- **X** and **Z** are horizontal axes.
- **Y** is the vertical axis (height).
- A Y value of about `2.0` places the spawn point one player-height above the floor.
- Leading/trailing whitespace and empty lines are ignored.

If the file is absent, four default spawn points are used (corners of the origin, Y = 2.0).

## Server Usage

```
./server [OPTIONS]

Options:
  --port <PORT>         UDP port for native clients         [default: 7777]
  --web-port <PORT>     WebTransport port for WASM clients  [default: 7778]
  --map-url <URL>       HTTPS URL of the map .tar.zst file  [optional]
```

### Example

```bash
./server --port 7777 --web-port 7778 \
         --map-url https://cdn.example.com/maps/dust2.tar.zst
```

When `--map-url` is omitted the server does **not** send a `MapUrlMsg`, and all clients continue to use the built-in placeholder map.

---

## Changing the Map at Runtime

Currently, the map URL is set once at server start.  Sending a new `MapUrlMsg` to all connected clients (e.g. at round end) is possible through the network protocol - see `src/network/protocol.rs` - but no automatic round-cycling logic is implemented yet.

---

## Hosting the Archive

Any standard HTTPS file server works.  Make sure to:

1. Serve with **Content-Type: application/octet-stream** (or any binary type).
2. Set **CORS headers** if clients are connecting from a browser (WASM build):
   ```
   Access-Control-Allow-Origin: *
   ```
3. Keep the file **reasonably small** (< 50 MB recommended).  
   Zstandard compression is very efficient for 3D assets - a typical small map  
   compresses to well under 10 MB.

## Authoring Checklist

**Blender scene preparation**
- [ ] All transforms applied (Ctrl+A → All Transforms) - no unapplied scale/rotation
- [ ] No negative scale on any object
- [ ] All meshes have UV map (UV channel 0)
- [ ] Normal map textures set to **Non-Color** color space in Blender
- [ ] Base Color / emissive textures set to **sRGB** color space in Blender
- [ ] At least one light in the scene (or ambient lighting configured)

**scene.glb export**
- [ ] Format: GLB (binary)
- [ ] Transform → Y Up: **On**
- [ ] Data → Mesh → Compression: **Off** (no Draco)
- [ ] Data → Mesh → Tangents: **On**
- [ ] Data → Mesh → Apply Modifiers: **On**
- [ ] Include → Punctual Lights: **On** (if using Blender lights)
- [ ] Data → Images → Export: **Automatic** (textures embedded) or files in `textures/`
- [ ] All floor/wall meshes are closed (no open edges - checked with Non-Manifold select)

**Archive and hosting**
- [ ] `spawn_points.txt` has at least 2 spawn locations, Y ≈ player height above floor
- [ ] Archive created with `tar` + `zstd` (NOT gzip or bzip2)
- [ ] File hosted over **HTTPS** and publicly accessible
- [ ] CORS header set if browser (WASM) clients will connect
- [ ] Tested by running `./server --map-url <your-url>` locally and connecting a client
